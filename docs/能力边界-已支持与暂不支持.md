# 能力边界：已支持与暂不支持

本文档单独列出 QA Test Framework 当前已支持能力和暂不支持能力。协议、运行方式或发布流程变化时，需要同步更新本文档、`docs/运行指南-启动构建与本地联调.md`、`docs/项目总览-架构流程与后续计划.md` 和对应模块 README；`agent.md` 只在整体架构、基础设定或文档导航变化时同步。

## 当前支持能力

### Register Server

- `registerserver/rustserver` 是主服务端实现，提供 HTTP API、WebSocket 协议、执行调度、请求序列、控制锁、执行历史和 Web 静态托管。
- `GET /api/health`、`GET /api/unity-clients`、`GET /api/results` 可用于健康检查、客户端列表和执行历史查询。
- WebSocket 支持 Unity 端 `register`、`heartbeat`、`qa_result`，以及 Web/MCP 端 `refresh`、`execute`、`execute_sequence`、`stop_execution`、`stop_sequence`。
- 支持 Text/Binary WebSocket payload 按 JSON 统一解析。
- 支持执行超时收敛、late result 广播、控制端锁、Unity 业务 heartbeat 不可用标记和 Unity 本地 busy 状态展示。
- 支持执行请求级 `timeoutMs`，HTTP `execute`、WebSocket `execute`、请求序列顶层和单个 step 均可覆盖默认 `EXECUTION_TIMEOUT_MS`。
- 支持 `allowParallelExecution`：当 Unity 注册方法或执行请求声明允许并行时，server 会透传该字段并放宽同一客户端的锁、running 和本地 busy 互斥拦截；该能力应仅用于只读查询类方法。
- 支持启动时自动加载 `registerserver/rustserver/.env`，并提供 `.env.example` 示例配置。
- 支持 `QA_LISTEN_HOST` 配置监听地址，默认 `0.0.0.0`；支持 `QA_ACCESS_SCOPE` 配置访问范围，默认 `private`，只允许本机、私网、链路本地和 IPv6 内网来源。
- 支持 `QA_CLIENT_DIST_DIR` 配置 Web 控制台构建产物目录；未设置时优先使用 exe 同目录下的 `client/dist`，本地开发回退到仓库中的 `registerserver/client/dist`。
- 支持 stdout JSON line 结构化日志，并可通过 `QA_LOG_DIR` 按天切分落地到文件；执行相关事件包含 `requestId`、`clientId`、`controllerId`、`sequenceId` 等关联字段。
- `/api/results` 和 Web 控制台执行历史保持最近 200 条内存视图；长期追溯可使用服务端 JSON line 日志文件，也可通过 `QA_EXECUTION_ARCHIVE_MYSQL_ENABLED` 开启旁路 MariaDB/MySQL 执行历史存档。
- 已有集中配置文档，说明端口、超时、token、heartbeat 和执行限制。
- `registerserver/server` 是冻结的 Node legacy 服务端，只作为历史行为参考和紧急本地回退入口，不再同步新能力、协议字段或常规修复。

### Web 控制台

- 展示在线 Unity 客户端、业务可用状态、本地 busy 状态、控制锁状态和注册方法列表。
- 支持执行单个 `[QaTest]` 方法。
- 支持请求序列、失败即停、步骤间隔 `stepDelayMs` 和执行历史查看。
- 构建后由 Rust server 托管 `QA_CLIENT_DIST_DIR` 指向的 Web 控制台构建产物。
- 支持可选 `QA_WEB_CONSOLE_TOKEN` 轻量登录；未配置时保持免登录，配置后浏览器控制台需要输入 token。

### qamcp

- 提供 stdio MCP tools：健康检查、客户端列表、方法搜索、方法详情、结果查询、单次执行、请求序列、停止执行、事件监听、等待结果和可轮询事件会话。
- 执行类工具通过 `WS /ws?role=web&controllerType=mcp&controllerId=...` 复用 Web 控制端协议。
- 支持通过 `qamcp config set serverUrl <url>` 指向目标 register server，默认 `http://localhost:3000`。
- 支持通过 `QA_CONTROLLER_ID` 固定 MCP 控制端身份。
- Windows 上支持通过 Rust release 构建 `dist/qamcp.exe`，MCP 客户端可直接把 `command` 指向 exe。
- qamcp 基础校验使用 `cargo fmt --check`、`cargo check` 和 `cargo test`。

### Unity Client

- Unity package 独立维护，当前仓库跟踪 `.meta` 文件；新增脚本必须同时提交 `.meta`，避免 GUID 漂移。
- 支持 QA 功能启用开关；Editor 默认启用，Player 默认关闭，避免正式包默认暴露 QA WebSocket 能力。
- 支持通过运行时 API、命令行参数、环境变量 `QA_TEST_ENABLED`、PlayerPrefs `QaTest.Enabled` 和 Inspector 默认值控制 QA 开关。
- 支持 `QaTestBootstrap` 在 QA 功能启用时自动创建 `QaTestClient`；Editor 进入 Play Mode 后默认自动连接 register server，Player 需要显式启用。
- 支持命令行参数、Inspector 字段和 PlayerPrefs 覆盖服务端地址。
- 支持 static class 静态 `[QaTest]` 方法、普通 class 静态 `[QaTest]` 方法，以及场景中已存在的 `MonoBehaviour` 实例 `[QaTest]` 方法。
- 注册给 server 的方法 `id` 当前使用短签名，例如 `Method(System.String)`；客户端内部保留包含声明类型和 Unity instance id 的 full id 用于缓存和实例目标查找。
- 支持参数转换：`string`、`bool`、整数、浮点数、枚举、`Vector2/3/4`、可空类型、可选参数、显式空字符串占位 `{"$empty":true}` 和可由 `JsonUtility.FromJson` 解析的类型。
- 支持返回普通值、`Task`、`Task<T>`、`IEnumerator`、`QaTestCoroutineReturn` 和 `QaTestCoroutineResult`。
- 支持业务失败语义兼容：`failed:<reason>`、`false`、`ok=false` 或失败态 `status` 的结构化 JSON 会映射为 `success=false`。
- 默认支持 Unity 本地单请求互斥；本地 busy 时会拒绝新的普通 `execute`。标记 `AllowParallelExecution` 的只读查询方法可并行执行。

## 暂不支持能力

- 暂无全局 HTTP/WebSocket token 鉴权、签名、细粒度白名单或角色权限模型；当前只提供默认内网来源限制和 Web 控制台登录 token，Unity 和 qamcp 在内网部署下仍保持免 token 调用。
- 服务端 `stop_execution`、`stop_sequence` 只能在 server 侧标记取消，暂不能强制中断 Unity 内部正在运行的方法。
- `/api/results` 不会从日志文件或 MariaDB/MySQL 回放恢复，服务重启后 Web/API 最近历史重新开始累计。
- `qa_result.result` 当前仍主要是字符串承载；虽已支持基础结构化业务失败判定，但尚未形成完整结构化测试结果、附件、截图、Unity log 或性能指标协议。
- Rust server 仍缺少协议状态机自动化测试，目前只覆盖了少量局部逻辑。
- 暂未抽出独立 JSON schema 或共享协议契约文件。
- 请求序列 step 校验仍需加强，避免错误输入被静默过滤。
- Node legacy 服务端已冻结；不要把它当作双主实现，也不要期待它跟随 Rust server 后续能力演进。
- Web 控制台仍需补充更完整的空状态、错误状态、连接恢复提示、方法收藏和参数模板。
