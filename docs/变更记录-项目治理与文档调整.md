# 变更记录：项目治理与文档调整

本文档记录 QA Test Framework 的整体项目治理、仓库结构和重要文档变化。

## 2026-06-11

- 仓库治理从 multi-repo/meta repo 调整为单一 Git 仓库，`qamcp`、`registerserver`、`unityclient` 源码由根仓库统一跟踪。
- 更新根 `.gitignore`，保留源码目录，忽略 Node/Rust 构建产物、本地配置和 Unity 生成目录。
- 同步 README、Agent 入口说明、发布矩阵和发布检查清单中的提交、推送和发布规则。

## 2026-05-19

- 对照当前代码修正 `docs/` 中 Unity package 版本、方法 ID、参数空字符串、业务失败判定、`timeoutMs`、`clientBusy` 和 `allowParallelExecution` 说明。
- 更新 `docs/发布版本矩阵-仓库Commit与协议版本.md` 当前开发组合 commit，记录 Unity package manifest 版本 `0.1.3`。
- 明确 `agent.md` 只维护入口级架构和文档导航，不再跟随协议字段细节同步。
- 合并项目入口类文档为 `docs/项目总览-架构流程与后续计划.md`，承载项目定位、架构、成果、核心流程和后续 Todo。
- 统一重命名 `docs/` 下专题文档为“主题-具体用途”格式，提升文件名可读性。
- 将原始需求移动到 `docs/archive/`，归档内容默认不再作为 Agent 当前任务读取来源。
- 删除 WholeClient 专项规划文档。

## 2026-05-08

- Unity package 版本号从 `0.1.1` 升级到 `0.1.2`。
- 更新 `docs/发布版本矩阵-仓库Commit与协议版本.md` 中 Unity client 组合 commit 为 `90ec2d2`。

- 更新 `docs/发布版本矩阵-仓库Commit与协议版本.md` 中 Unity client 组合 commit 为 `70b4227`。
- 新增 `docs/能力边界-已支持与暂不支持.md`，集中维护当前支持能力和暂不支持能力。
- 修正 Unity package `.meta` 策略文档，明确 package 仓库必须跟踪 `.meta` 文件。
- 同步 qamcp 文档，默认使用 Rust 主服务端，并补充 `dist/qamcp.exe` 交付方式。
- 同步 `agent.md` 的 Rust server 模块拆分、qamcp exe 和文档同步规则。
- 更新 `docs/发布版本矩阵-仓库Commit与协议版本.md` 当前开发组合 commit。
