# 运行指南：启动、构建与本地联调

整体架构、模块职责和 Web/MCP/Unity 交互链路见：[项目总览-架构流程与后续计划](./项目总览-架构流程与后续计划.md)。

当前已支持能力和暂不支持能力见：[能力边界-已支持与暂不支持](./能力边界-已支持与暂不支持.md)。

端口、超时、token、heartbeat 和执行限制见：[配置参考-端口超时鉴权与执行限制](./配置参考-端口超时鉴权与执行限制.md)。

## 目录

- `registerserver/rustserver`: Rust + Axum + Tokio 主服务端
- `registerserver/server`: Node.js + Express + WebSocket 冻结 legacy 服务端，仅作历史参考和紧急回退
- `registerserver/client`: Vue 3 + Element Plus 网页控制台
- `qamcp`: stdio MCP 工具服务，供 AI Agent 调用 QA 能力
- `unityclient/My project`: Unity 2022.3 工程和运行时客户端

## 启动服务端

Register server 的完整部署、HTTP API、WebSocket 协议和排障说明见：[RegisterServer-服务端 API 与 WebSocket 协议](./RegisterServer-服务端API与WebSocket协议.md)。
默认只启动 Rust 主服务端。Node legacy 已冻结，不参与常规运行；只有紧急回退时才参考：[NodeLegacy 服务端-冻结规则与回退指南](./NodeLegacy服务端-冻结规则与回退指南.md)。

```powershell
cd registerserver/rustserver
cargo run
```

默认地址是 `http://localhost:3000`。服务端会同时提供：

- `GET /api/health`: 健康检查
- `GET /api/unity-clients`: 当前在线 Unity 实例
- `GET /api/results`: 最近执行记录
- `POST /api/unity-clients/:clientId/execute`: 下发执行指令
- `WS /ws?role=unity`: Unity 长连接
- `WS /ws?role=web`: 网页控制台长连接，支持单次执行和请求序列执行

默认配置面向内网测试：服务端监听 `0.0.0.0:3000`，但只允许本机、私网和链路本地来源访问。需要取消来源限制时，在 `.env` 或启动环境中显式设置 `QA_ACCESS_SCOPE=unrestricted`。

内网部署时如需给网页控制台加登录 token，可在启动服务端前设置：

```powershell
$env:QA_WEB_CONSOLE_TOKEN="replace-with-internal-token"
cargo run
```

未设置 `QA_WEB_CONSOLE_TOKEN` 时网页控制台保持免登录。该 token 只影响浏览器控制台登录；Unity 客户端和 qamcp 调用方式不变。

需要集中配置时，可复制并编辑本地 `.env`：

```powershell
cd registerserver/rustserver
Copy-Item .env.example .env
cargo run
```

`.env` 会被 git 忽略，配置项说明见：[配置参考-端口超时鉴权与执行限制](./配置参考-端口超时鉴权与执行限制.md)。

## 启动网页控制台

生产方式：

```powershell
cd registerserver/client
npm install
npm run build
```

构建后由服务端直接托管：

- `http://localhost:3000/`: 测试控制台
- `http://localhost:3000/sequences`: 请求序列
- `http://localhost:3000/history`: 执行记录

开发方式：

```powershell
cd registerserver/client
npm run dev
```

开发地址默认是 `http://localhost:5173`，Vite 会把 `/api` 和 `/ws` 代理到 `http://localhost:3000`。

## 启动 AI MCP 工具

`qamcp` 会把当前 Web 控制台支持的 QA 能力包装成 MCP tools。读操作通过 server HTTP API 完成，执行操作通过 `WS /ws?role=web` 完成。

常规 server、web、Unity 三端运行不依赖 MCP。只有需要让 AI Agent 调用 QA 能力时，才需要配置 `qamcp`。

MCP 的安装、客户端配置、tools 列表和 curl 调试命令已拆分到独立文档：[QAMCP-Agent 工具使用指南](./QAMCP-Agent工具使用指南.md)。

对外接入 MCP 时，优先使用 Rust 构建产物 `qamcp/dist/qamcp.exe`。

## 请求序列

Web 控制台的“请求序列”页面会通过 WebSocket 向 server 提交一组步骤：

```json
{
  "type": "execute_sequence",
  "sequenceId": "uuid",
  "clientId": "unity-client-id",
  "stopOnFailure": true,
  "stepDelayMs": 1000,
  "steps": [
    {
      "stepId": "uuid",
      "methodId": "Ping()",
      "methodName": "连通性检查",
      "arguments": []
    }
  ]
}
```

`stepDelayMs` 表示相邻步骤之间的等待时间，单位毫秒，范围是 `0` 到 `300000`。server 会按顺序把每个步骤转发给 Unity，等待 Unity 的 `qa_result` 后，再按 `stepDelayMs` 等待并执行下一步。Web 端会收到：

- `sequence_started`: 序列开始
- `sequence_step_started`: 单步开始
- `sequence_step_result`: 单步返回
- `sequence_finished`: 序列完成和汇总

## 启动 Unity 客户端

Unity client 的接入方式、`[QaTest]` 写法、参数转换、返回值和排障说明见：[UnityClient-接入 QaTest 与运行时行为](./UnityClient-接入QaTest与运行时行为.md)。

用 Unity 2022.3.62f1 打开 `unityclient/My project`，进入 Play Mode。Editor 下 QA 功能默认启用，会自动创建 `[QaTestClient]` 对象并连接：

```text
ws://localhost:3000/ws?role=unity
```

不需要手动改场景。需要连接其他服务器时，可以用启动参数覆盖：

```powershell
--qa-server-url ws://服务器地址:3000/ws
```

也可以在运行时写入：

```csharp
PlayerPrefs.SetString("QaTest.ServerUrl", "ws://服务器地址:3000/ws");
PlayerPrefs.Save();
```

构建后的 Player 默认关闭 QA 功能。测试包需要显式开启：

```powershell
Game.exe --qa-test-enable --qa-server-url ws://服务器地址:3000/ws
```

也可以用环境变量、PlayerPrefs 或运行时 API 控制：

```csharp
QaTestClient.SetGlobalEnabled(true);
PlayerPrefs.SetInt(QaTestClient.EnabledPlayerPrefsKey, 1);
PlayerPrefs.Save();
```

## 编写 QaTest 方法

引用命名空间后给方法加 `[QaTest]`：

```csharp
using QaTestFramework;

public class LoginSmoke
{
    [QaTest("Ping")]
    private static string Ping()
    {
        return "pong";
    }
}
```

支持静态方法，以及场景中已存在的 `MonoBehaviour` 实例方法。参数会从网页输入的字符串转换为常用类型：`string`、`bool`、整数、浮点数、枚举、`Vector2/3/4`。返回值会回传到网页执行记录。

框架底层脚本放在 `unityclient/My project/Assets/scripts/Runtime/QaTest`。示例或上层测试方法放在 `unityclient/My project/Assets/scripts/Samples~/Example`，例如：

- `QaTestSample.cs`: 基础 ping、日志、等待返回示例
- `QaTestPanel.cs`: GameObject/面板存在性、显隐、等待状态
- `QaTestControl.cs`: Unity UI Button/Selectable 点击、可交互状态、等待状态

协程可以返回结果。普通 `IEnumerator` 中可以：

```csharp
yield return QaTestCoroutineReturn.From("done");
```

需要用包装对象时可以返回 `QaTestCoroutineResult`，由框架等待协程结束后读取结果工厂的返回值。
